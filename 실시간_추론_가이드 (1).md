# 업비트 AI 실시간 추론 가이드 (ForKang)

## 📋 개요
이 패키지는 업비트 API를 사용하여 **실시간 1분봉 추세를 예측**하는 XGBoost 모델입니다.

### 주요 특징
- **모델**: XGBoost Gradient Boosting (CUDA 학습, CPU 추론)
- **예측 대상**: 30분 뒤 수익률이 12bp 이상인지 여부 (수수료 5bp 차감 후)
- **입력**: 업비트 1분봉 OHLCV + 바이낸스 동시점 데이터
- **출력**: 0~1 사이의 확률 점수 (임계값 이상 매수 신호)
- **지연**: 20종목 기준 약 3~5ms

## 📊 모델 성능

### 학습 통계
- **학습 기간**: 최근 4년 (2022-2026)
- **사용 마켓**: 업비트 top20 코인
- **총 학습 샘플**: ~2M 개 (최신 데이터에 65% 가중)
- **양성 비율**: ~15% (수익 기회 희소)

### 검증 성능 (Test 기준)
- **Average Precision (AP)**: ~0.65-0.75
- **ROC-AUC**: ~0.70-0.80
- **Hit Rate** (선택한 거래만): ~50-60%
- **평균 거래 수익**: 비용 차감 후 +0.8~2.0% (한 거래 기준)

### 실제 거래 관점
```
예상 시나리오:
- 1000 거래 중 약 200-300 거래가 신호 발생
- 그 중 약 100-150 거래 성공 (50-60% hit)
- 1 거래 평균 +0.8~2.0%, 손실 거래 -1.0~2.5%
- 평균 월 수익: 변동성 높음 (back-test 기준)
```

**⚠️ 주의**: 이전 성과가 미래를 보장하지 않습니다. 실거래 전 충분한 검증이 필요합니다.

## 🔧 피처 설명

### 5대 피처 그룹

#### 1. 가격 모멘텀 (6개)
- `ret_1m ~ ret_2880m`: 지난 1분~48시간 수익률
- 단기 추세 포착

#### 2. 캔들 구조 (4개)
- `range_pct`: 일중 범위 (고가-저가) / 종가
- `body_pct`: 실체 (종가-시가) / 범위
- `upper_wick_pct`, `lower_wick_pct`: 윗꼬리, 아래꼬리 비율
- 캔들 형태 분석

#### 3. 변동성 (10개)
- `volatility_*m`: 지난 N분 표준편차
- `realized_vol_*m`: 지난 N분 일일 변동성 (Parkinson)
- 시장 불안정성 측정

#### 4. 추세 지표 (9개)
- `ema_*_ratio`: EMA 단기/장기 괴리
- `macd_*`: MACD ratio, signal, histogram
- `rsi_14`: 상대강도지수
- `bb_z_20`: 볼린저 밴드 z-score
- `dist_high_60`, `dist_low_60`: 60분 고/저가 거리
- `breakout_20`: 20분 신고가 돌파 여부

#### 5. 시장 구조 (30개+)
- **유동성**: 거래량/거래대금 상대 비율, z-score
- **크로스마켓**: 바이낸스 동시점 수익률, 거래량
- **스프레드**: 업비트-바이낸스 수익률 차이, 김프, 김프 변화율
- **거시**: 환율, BTC 도미넌스, BTC/ETH 추세
- **시간**: 일중 시간, 요일 (sin/cos 인코딩)

})

# 모델 로드

    model = pickle.load(f)

# 1분마다 신호 생성

while True:
    try:
        markets = ['KRW-BTC', 'KRW-ETH', 'KRW-XRP', ...]  # 관심 마켓
        
        for market in markets:
            # Upbit 1분봉 조회 (최근 120개)
            ohlcv = upbit.fetch_ohlcv(market, '1m', limit=120)
            
            # Binance 동시점 데이터 조회 (별도 로직 필요)
            binance_ohlcv = get_binance_ohlcv(market, '1m', len(ohlcv))

### 예제 2: 배치 추론 (백테스트)
```python
# 모든 마켓, 모든 시점 데이터 로드
data = pd.read_csv('historical_1m.csv')  # 마켓, 시간, OHLCV, 바이낸스, 환율

# 피처 생성
features = build_realtime_features(data, config=model.config, include_target=False)

# 신호 생성
signals = model.predict_signal(features)

# 거래 로직
trades = []
for _, signal_row in signals.iterrows():
    if signal_row['signal'] == 1:
        # 30분 뒤 실제 수익률 조회
        future_price = get_future_price(
            signal_row['market'],
            signal_row['timestamp_utc'],
            minutes=30
        )
        actual_return = (future_price - entry_price) / entry_price - 0.05  # 5bp 비용
        trades.append({
            'market': signal_row['market'],
            'entry': signal_row['timestamp_utc'],
            'prob': signal_row['trend_probability'],
            'return': actual_return
        })

# 성과 분석
import numpy as np
returns = [t['return'] for t in trades]
print(f"총 거래: {len(returns)}")
print(f"승률: {(np.array(returns) > 0).mean():.2%}")
print(f"평균 수익: {np.mean(returns):.2%}")
print(f"Sharpe: {np.mean(returns) / np.std(returns) * np.sqrt(252*6.5*60):.2f}")  # 1분 기준
```

## ⚠️ 주의사항

### 1. 데이터 정확성
- **완성된 캔들만 사용**: 1분 캔들이 완전히 종료된 후에만 추론
- **진행 중인 캔들 제외**: 마지막 미완성 캔들은 NaN 또는 제거
- **타이밍**: Upbit 1분 캔들 종료 시점(xx:xx:00) 직후 추론

### 2. 바이낸스 데이터 부재
- 일부 마켓은 바이낸스에 없을 수 있음
- → 해당 피처 `NaN` 처리 (wrapper가 보정)
- → 성능 저하 가능 (5~10%)

### 3. 환율 및 김프
- 동시점 환율 필수 (차익거래 신호)
- 김프 부정확 시 거래 수익 왜곡
- 실거래 전 데이터 정확성 검증 필수

### 4. 네트워크 지연 & API 한계
- Upbit API 분당 요청 제한 (tier별 상이)
- 여러 마켓 동시 조회 시 배치 처리 권장
- 바이낸스 데이터 동기화 지연 대비 필요

### 5. 거래 비용
- **Upbit 현물 수수료**: 0.05% (일반)
  - 매수 0.05%, 매도 0.05% = 총 0.10% (10bp)
- 모델은 5bp 차감 기준으로 학습했으므로, 실거래는 더 보수적
- **슬리피지**: 호가 잔량, 주문량에 따라 추가 비용

### 6. 모델 한계
- **학습 기간 편향**: 2022~2026 강세장(부분) 기반
  - 극심한 약세 또는 변동성 급증 시 성능 저하 가능
- **과거 상관 기반**: 시장 구조 변화에 천천히 적응
- **라벨 정의 변경**: 미래 수익률이 12bp 이상인 경우로 학습
  - 실제 거래는 손실 가능성 항상 존재

## 📞 기술 지원 체크리스트

모델 로드 오류 발생 시:
```
1. pickle 버전 확인: python -c "import pickle; print(pickle.__version__)"
2. XGBoost 버전 확인: pip show xgboost
3. NumPy/Pandas 호환성: pip install --upgrade numpy pandas
4. CUDA 드라이버: nvidia-smi (추론 CPU에서 실행하므로 선택)
```

피처 생성 오류 시:
```
1. 입력 컬럼 확인: set(df.columns) 와 model.feature_columns 비교
2. 타임스탠프 형식: pd.to_datetime() 변환
3. NaN 처리: df.fillna(0) 또는 df.ffill()
```

신호 생성 오류 시:
```
1. 모델 로드 성공 확인: model.model, model.threshold 존재 여부
2. 입력 데이터 형태: signals.shape, 마켓 수
3. threshold 값: model.threshold (기본 ~0.5)
```
